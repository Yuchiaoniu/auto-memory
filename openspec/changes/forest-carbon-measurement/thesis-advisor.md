## 二、DSR 六步驟對應本系統

```
步驟 1：問題辨識與動機
  → 人工測量 DBH 耗時耗力，碳匯數據缺乏可信存證機制

步驟 2：定義目標
  → 手機影片 → AI估算DBH → 區塊鏈存證，全流程自動化

步驟 3：設計與開發（主體）
  → 支柱一：AI量測管道（薄透鏡+參照物雙路徑）
  → 支柱二：修正因子自演進（加權平均 + 事件驅動快照）
  → 支柱三：Besu 不可篡改存證（CarbonCredit.sol）

步驟 4：展示
  → 實地測量 20-30 棵樹，上傳影片，驗證全流程可跑通

步驟 5：評估
  → MAE、MAPE、RMSE、R²（與人工量測比較）
  → Bootstrap 95% CI（小樣本統計嚴謹性）
  → DSR checklist 七項準則評分

步驟 6：溝通
  → 畢業專題報告（中文）+ MDPI Forests 投稿（英文）
```

## 七、寫作注意事項

1. **每個 RQ 在第一章提出，在第四章有對應數字回答**，不能有懸空的 RQ
2. **系統描述必須用被動語態**（英文版），如 "frames are extracted" 而非 "we extract frames"
3. **數字要有單位與精度**，如 "MAE = 2.3 ± 0.4 cm (95% CI)"
4. **中文版圖表標題在圖下方**；英文版（MDPI）圖表標題格式依官方 template
5. **不要在方法章節（第三章）出現評估結果**，評估全部放第四章
6. **第三章要有系統架構圖**（三支柱 + 資料流向），這是審查委員最先看的東西
7. **引用 DSR 框架時**，明確說明你的系統屬於 Instantiation（實例化），不是 Construct 或 Model
8. **積極情緒螺旋**：行文語氣應朝向「此系統能做到什麼」推進，而非放大問題的痛苦感。每一段的情緒走向應是遞進式的正向積累，而非先壓低再拉高。
9. **一氣呵成**：同一段落內盡量維持單一語氣方向，避免頻繁使用「而」、「但是」等轉折詞打斷語流。若需對比，改用「相較之下」、「進一步」、「同時」等順向連接。
10. **避免恐懼行銷**：不以負面描述堆疊來製造問題感。例如「操作者需在現場環繞樹幹量測，費時費力且高度依賴操作員經驗」屬於恐懼行銷；應改為強調新方法帶來的具體改善，如「此系統以手機影片取代現場環繞量測，降低對操作員經驗的依賴」。
11. **避免語氣強烈的詞彙**：如「強制」、「完全」、「絕對」、「必須」等帶有命令或絕對意涵的詞彙，改以中性描述取代。例如「強制設為 high」→「自動判斷為 high」；「完全繞開」→「無需依賴」。

## 八、中文改寫句型規則

### 規則一：避免「X 是 Y」短句串聯，改用長句形容詞結構
- 將「X 是 Y」等號句型改寫為「以 [功能/關係描述] 修飾的 [主體]，[動詞片語]…」的長句，把定義或性質變成形容詞修飾語嵌入句中。
- **反例**：「DBH 是林業碳匯估算中最關鍵的單木測量指標，定義為距地面 1.3 公尺高度處的樹幹直徑。」
- **正例**：「透過…異速生長方程式計算可直接轉換為材積且與樹木生物量成高度相關的 DBH，定義為距地面 1.3 公尺高度處的胸高樹幹直徑，可以是林業碳匯估算中關鍵的單木測量指標。」

### 規則二：避免絕對陳述，改用適度語氣
- 「最關鍵」→「關鍵」；「是…基礎」→「成為…基礎」或「可以是…基礎」。
- 學術寫作應保留語氣空間，避免無來源支撐的絕對性主張。

### 規則三：資訊排序以功能／機制優先，定義置後
- 先說明概念的用途或計算機制，再給出定義，強調應用脈絡而非純粹名詞解釋。

### 規則四：技術術語加英文對照
- 重要技術術語首次出現時，在括號內附上英文全稱或縮寫，必要時加英文說明（如 Reliability）。

### 規則五：用詞規範
- 全文一律使用「此專案」，不使用「本研究」。
- 中文句子禁止以「的」結尾，改用完整謂語結構。
- 「閉環」一律改為「循環」。

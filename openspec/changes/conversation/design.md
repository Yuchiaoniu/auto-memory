## Context

目標使用者是自閉症者（ASD）及其照護者或教師。現有的社交技能訓練多依賴治療師一對一進行，資源有限且難以在日常生活中隨時練習。本專案以純靜態網頁實作，讓使用者可在家自主練習，不需要帳號、不需要網路連線，降低使用門檻。

## Goals / Non-Goals

**Goals:**
- 提供低壓力的情境式對話練習環境
- 三難度層次漸進設計（複誦 → 選擇 → 自由輸入），避免使用者一開始就感到挫折
- 回應提示系統幫助使用者在卡關時找到出口，而非強迫自行想出答案
- 進度記錄讓使用者（與照護者）看見累積練習量
- 純靜態部署，可放在 GitHub Pages，零維護成本

**Non-Goals:**
- 不做語音辨識或 TTS（降低複雜度，聚焦文字練習）
- 不做使用者帳號系統（localStorage 足夠個人裝置使用）
- 不做 AI 即時生成回應（避免不可預期輸出讓使用者更困惑）
- 不涵蓋非日常生活的特殊情境（醫療面談、法律場合等）

## Decisions

**決策 1：純靜態 HTML/JS，不使用框架**
- 選擇原因：部署到 GitHub Pages 不需要建置步驟，維護者不需要 Node.js 環境
- 考慮過 React/Vue：對於內容固定的練習型工具，框架帶來的複雜度大於收益
- 情境資料與對話腳本以 JSON 檔儲存，便於日後新增情境而不動 JS 邏輯

**決策 2：三難度層次設計**
- 複誦（Repeat）：顯示完整句子，使用者照著打一遍或按「我說了」確認
- 選擇（Choose）：顯示 2–3 個回應選項，使用者點選最適合的
- 自由輸入（Free）：顯示情境，使用者自行輸入回應，完成後顯示參考答案
- 設計邏輯：從「沒有壓力」到「有壓力」漸進，照護者可根據使用者程度鎖定特定層次

**決策 3：進度資料存於 localStorage**
- 每個情境記錄「完成次數」與「最後練習日期」
- 不使用 IndexedDB（過度設計）
- 資料格式：`{ scenarioId: { count: N, lastDate: "YYYY-MM-DD" } }`

**決策 4：回應提示以「提示卡」形式呈現**
- 提示預設隱藏，使用者主動點「給我提示」才出現，避免直接給答案降低練習效果
- 每個情境配備 2–3 張提示卡，措辭簡單、口語化

## Risks / Trade-offs

- [風險] localStorage 在無痕模式或跨裝置無法保留進度 → 接受此限制，說明文件中標注
- [風險] 情境腳本若措辭不夠自然，可能給使用者錯誤示範 → 由照護者或語言治療師審閱初版情境後再發布
- [取捨] 不做語音輸出，純文字介面對部分 ASD 使用者可能較難連結到實際對話 → 未來可用 Web Speech API 擴充，現階段先驗證核心流程

# Conversation 專案記憶庫

---

## L1 框架：衡量 → 配置 → 獲取（借自 exercise 專案行為科學模型）

對話的第一層閉環，對應三個生化訊號：

| 階段 | 對話意義 | 對應激素 |
|------|----------|---------|
| **衡量** | 評估場景：對方是誰、情緒溫度、社交安全感、是否有威脅 | 皮質醇（壓力偵測）|
| **配置** | 決定投入方向：說什麼、用什麼語氣、目標是連結還是資訊 | 多巴胺（動機信號）|
| **獲取** | 執行對話行為，收穫社交報酬（連結感、被理解感） | 睪固酮（完成訊號）|

閉環邏輯：衡量→配置→獲取→重新衡量（每一輪回應都重跑一次）

關鍵約束：皮質醇是支配變數——社交威脅感高時，配置與獲取兩個階段的資源都會被壓縮。

## L4 情境層：有旁觀者在場的對話場景

### 三元視角（對話從二元變三元）

私下對話是二元結構（你 ↔ 對方）。有旁觀者在場時，出現三條線同時運作：

- **線 A（內容層）**：你 ↔ 對方，說什麼、怎麼說
- **線 B（形象層）**：你 → 旁觀者，旁觀者如何詮釋你
- **線 C（後設層）**：你猜對方在想旁觀者的什麼

認知負擔來源：B 和 C 同時運作會佔用工作記憶，擠壓管理 A 的資源。
常見失誤：只顧 A（內容對了，形象輸了）或只顧 B（表演給旁觀者看，失去真實連結）。

### 旁觀者類型分類

| 類型 | 特徵 | 行動空間 |
|------|------|---------|
| 支持型 | 朋友在場，可能幫你 | 大，可借力 |
| 中立型 | 陌生人，不介入 | 中，不干擾也不幫 |
| 敵意型 | 可能加入攻擊或起鬨 | 小，避免給藉口 |
| 權威型 | 老師、主管在看 | 中，遵循場合規範 |

### 面子動態（Face Dynamics）

面子是動態餘額，不是固定資產。在旁觀者面前對話時：

```
面子餘額
  高 │    ╭─────╮
     │   ╱       ╲
  中 │──╱─────────╲──→ 理想穩態（對話順暢）
     │             ╲
  低 │              ╲── 危險區（觸發防衛反應）
     └──────────────────→ 對話進程
```

攻擊型對方的常見策略：快速把對方面子餘額打到低點，誘發防衛反應。
旁觀者只看到防衛反應，記不住攻擊起點——這是公開羞辱的機制核心。

**三種高風險場景：**
1. 公開羞辱型攻擊：對方借旁觀者做武器（「大家都看到了」）
2. 表演型衝突：對方其實在對旁觀者表演，不是真的在跟你對話
3. 旁觀者突然介入：改變對話的權力結構，需要重新定位角色

## 設計決策記錄

- 這個專案是心智探討/框架設計，不是網頁工具
- L1 框架直接借用 exercise 專案的行為科學三階段（衡量>配置>獲取）
- L4 情境層繼承 psych-resilience 專案的旁觀者視角邏輯，但從說話者角度重構
- psych-resilience 的 intervention-tree 是「旁觀者如何介入」；本專案翻轉為「說話者在旁觀者面前如何管理對話」

## 1. 專案骨架

- [ ] 1.1 建立資料夾結構：index.html、css/style.css、js/app.js、data/scenarios.json
- [ ] 1.2 建立 scenarios.json，包含至少六個日常情境（打招呼、請求幫忙、日常購物、課堂互動、問路、加入話題）
- [ ] 1.3 每個情境包含：id、title、category、description、steps（含三難度的對話步驟與提示卡）

## 2. 情境瀏覽器（scenario-browser）

- [ ] 2.1 實作首頁情境卡片列表，從 scenarios.json 動態渲染
- [ ] 2.2 實作分類標籤篩選（點選標籤只顯示該分類情境）
- [ ] 2.3 點擊情境卡片後切換到練習介面

## 3. 引導練習流程（guided-practice）

- [ ] 3.1 實作難度選擇畫面（複誦／選擇／自由輸入三個按鈕）
- [ ] 3.2 實作複誦層次：顯示示範句 + 「我說了」確認按鈕 + 進入下一步邏輯
- [ ] 3.3 實作選擇層次：顯示 2–3 個選項 + 點選後回饋訊息 + 進入下一步邏輯
- [ ] 3.4 實作自由輸入層次：文字輸入框 + 送出後顯示參考答案 + 跳過按鈕
- [ ] 3.5 實作練習完成畫面：鼓勵訊息 + 「再練一次」+ 「選其他情境」按鈕

## 4. 回應提示系統（response-hints）

- [ ] 4.1 在每個練習步驟加入「給我提示」按鈕，預設隱藏提示內容
- [ ] 4.2 點擊後展開提示卡，顯示 2–3 個口語化語句範本
- [ ] 4.3 確認使用提示不影響進度計數邏輯

## 5. 進度追蹤（progress-tracker）

- [ ] 5.1 實作 localStorage 讀寫工具函式（saveProgress、loadProgress、resetProgress）
- [ ] 5.2 在練習完成時呼叫 saveProgress 更新計數與日期
- [ ] 5.3 建立進度頁面，列出所有情境的練習次數與最後練習日期
- [ ] 5.4 實作「重置所有進度」按鈕，含確認對話框

## 6. 導覽與樣式

- [ ] 6.1 實作頂部導覽列（首頁 / 我的進度）
- [ ] 6.2 套用基本 CSS 樣式：字體放大、高對比色彩、大按鈕（適合 ASD 使用者閱讀）
- [ ] 6.3 確認在手機與桌面都能正常顯示（RWD）

## 7. 部署

- [ ] 7.1 建立 GitHub repo，設定 GitHub Pages（main 分支根目錄）

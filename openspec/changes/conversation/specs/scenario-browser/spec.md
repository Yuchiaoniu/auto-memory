## ADDED Requirements

### Requirement: 情境清單顯示
系統 SHALL 在首頁顯示所有可練習的對話情境，每個情境包含標題、情境說明文字與難度標示。

#### Scenario: 首頁載入情境清單
- **WHEN** 使用者開啟網站首頁
- **THEN** 系統顯示至少六個情境卡片，每張卡片含情境名稱與一句說明

#### Scenario: 情境依分類篩選
- **WHEN** 使用者點選分類標籤（如「打招呼」「請求幫忙」「日常購物」）
- **THEN** 系統只顯示該分類底下的情境卡片

### Requirement: 情境資料以 JSON 管理
系統 SHALL 從獨立的 JSON 資料檔讀取情境清單，不將情境內容硬編碼於 HTML 或 JS 中。

#### Scenario: 新增情境不需改動邏輯程式碼
- **WHEN** 開發者在 JSON 資料檔新增一個情境物件
- **THEN** 網站重新載入後該情境自動出現在清單中

### Requirement: 情境選擇進入練習
系統 SHALL 在使用者點擊情境卡片後，進入該情境的引導練習流程。

#### Scenario: 點擊情境卡片
- **WHEN** 使用者點擊某個情境卡片
- **THEN** 系統切換到該情境的練習介面，顯示第一個對話步驟

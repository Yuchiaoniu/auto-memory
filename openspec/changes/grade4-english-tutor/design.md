## Context

數位學伴計畫中，大學伴與四年級學生透過視訊一對一進行線上陪讀。目前缺乏共用的互動式教材平台，需要一個可即時共用螢幕、無需登入、快速切換單元的靜態網站。內容依台灣教育部英語課綱設計（不複製翰林課本），以 8 大主題單元為骨幹。

## Goals / Non-Goals

**Goals:**
- 純靜態網站，無後端，可部署至 GitHub Pages
- 涵蓋 8 單元完整教材（單字、句型、對話、測驗）
- 大學伴有專屬提示面板，學生看不到（摺疊/隱藏切換）
- 適合桌機共用螢幕使用，兼容平板

**Non-Goals:**
- 不做學生帳號、學習歷程追蹤（無後端）
- 不做音檔播放（避免版權與製作成本）
- 不做多語系或其他年級

## Decisions

**D1：純靜態 HTML/CSS/JS，不使用前端框架**
- 理由：部署成本零（GitHub Pages），無需 build 步驟，大學伴可直接用瀏覽器開啟
- 替代方案：Vue 3（被排除，因 build 流程對非技術大學伴不友善）

**D2：教材資料以 JS 物件陣列硬編碼**
- 理由：無後端，省去 API 設計；未來若要擴充可直接編輯資料檔
- 替代方案：JSON fetch（被排除，需要 HTTP server 才能本地開啟）

**D3：8 單元主題依教育部課綱設計**
```
Unit 1: Hello, Friends!（問候與自我介紹）
Unit 2: My Family（家庭成員）
Unit 3: At School（學校生活與教室物品）
Unit 4: I'm Hungry!（食物與飲料）
Unit 5: Animals I Like（動物）
Unit 6: Dress Up!（顏色與衣物）
Unit 7: What Time Is It?（數字與時間）
Unit 8: What's the Weather?（天氣與活動）
```

**D4：大學伴提示面板預設收起，點擊「大學伴模式」展開**
- 理由：避免學生看到答案提示，陪讀時大學伴自行決定是否展開
- 替代方案：獨立頁面（被排除，切換不便）

## Risks / Trade-offs

- [無法追蹤學習進度] → 大學伴需自行記錄；未來若需要可接 localStorage
- [單字無音檔] → 大學伴口頭帶讀；可考慮未來用 Web Speech API 合成語音
- [內容與實際課本進度不完全一致] → 明確標示「依課綱設計，非翰林課本複製」
- [測驗答案前端可見] → 本工具定位為練習而非考試，接受此限制

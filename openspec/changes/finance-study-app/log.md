## 如果要部署到 GCP，技術選項是這樣
```
┌─────────────────────────────────────────────────────┐
│              GCP 部署選項比較                        │
├──────────────┬──────────────┬────────────────────────┤
│ 選項         │ 難度         │ 適合情境               │
├──────────────┼──────────────┼────────────────────────┤
│ Cloud Run    │ 中（需 Docker│ 最推薦：按量計費，      │
│              │ Dockerfile） │ 閒置時不收費            │
├──────────────┼──────────────┼────────────────────────┤
│ App Engine   │ 低（直接     │ 最簡單，但 SQLite 要    │
│ Standard     │ gcloud deploy│ 換成 Cloud SQL 或       │
│              │）           │ Firestore               │
├──────────────┼──────────────┼────────────────────────┤
│ Compute Engine│ 低（跟現有  │ 你已有 Besu VM 環境，   │
│ VM           │ Besu 一樣）  │ 可以直接 SSH 進去跑     │
└──────────────┴──────────────┴────────────────────────┘
```
**SQLite 的問題**：Cloud Run / App Engine 的容器是無狀態的，每次重啟資料會清空。如果要在雲端保留題庫資料，有三條路：
```
路線 1：把 SQLite 資料庫打包進 Docker image
        → 資料固定，不會清空，但也無法在雲端寫入（quiz_results 不能存）
        → 適合：只用來查筆記 + 練習題，不在意成績記錄
路線 2：用 Cloud SQL（PostgreSQL / MySQL）取代 SQLite
        → 需要改寫 database.js
        → 適合：想要真正的雲端多人使用
路線 3：部署到 Compute Engine VM（你現有的 GCP 環境）
        → 跟跑 Besu 節點一樣的方式 SSH 上去跑 npm start
        → SQLite 檔案存在 VM 磁碟，不會消失
        → 最簡單，但 VM 要一直開著（有費用）
```
---
## 想釐清的方向
幾個問題可以幫助決定方向：
1. **手機 / 平板複習**需求有多強？還是筆電就夠用了？
2. **quiz_results 成績記錄**重不重要？需要在雲端保存歷史嗎？
3. 想先**只推 GitHub**（原始碼備份），還是同時要公開 URL？
4. 現有的 Besu VM 還有多少剩餘空間可以用？
這四個問題的答案會決定是「只推 GitHub」還是「加上 GCP 部署」，以及用哪種部署架構。

## 改名討論：稀缺域 → 分配域
這個改動有一個非常精準的直覺：
```
稀缺域 描述的是「事實」   → 資本不夠用（被動約束）
分配域 描述的是「行為」   → 你必須決定誰得到什麼（主動回應）
```
如果財務管理是行為科學，那麼「分配」比「稀缺」更接近行為主體的視角。  
稀缺是環境給你的，分配是你對環境的回應。**這個改動讓第一層從「約束描述」變成「行為描述」。**
---
## 改名討論：不確定域 + 量測域 → 確認域
這裡有一個值得慢慢想的張力：
```
H2 不確定域：針對「未來」——我們不知道接下來會發生什麼
H4 量測域：  針對「現在」——我們必須花力氣才能看清楚現況
```
兩個「不知道」的性質不同：
```
H2 的不知道 → 未來根本不存在，無從知道
            → 回應方式：風險溢酬、β 值、加碼折現率
H4 的不知道 → 現在存在，但需要工具才能看見
            → 回應方式：財務報表、比率分析、現金流追蹤
```
「確認域」把兩者合在一起，背後的邏輯是：
> **在做決策之前，你必須先確認「現在在哪裡」和「未來可能在哪裡」，這兩件事都是「確認現實」的行為。**
這個邏輯是成立的。問題是工具差太遠，考試出題也分屬兩個世界。  
但如果第一層的目的是「最廣義的行為類別」，而不是「給章節分類的箱子」，那合併是有意義的。

## 更大的問題：財務管理是行為科學嗎？
先問一個更根本的問題：
```
傳統財務的自我定位：
  → 工程學（公式、最佳化）
  → 經濟學（理性代理人、均衡）
行為財務學的主張：
  → 人類做財務決策時有系統性偏誤
  → 市場不完全理性
你的問題比這個更深：
  → 財務管理的「最廣義定義」是不是
     「組織（或個人）如何對現實約束做出回應的行為科學」？
```
如果是的話，那麼所有財務工具都是「行為回應的技術包裝」：
- CAPM 不是數學公式，而是「人類面對不確定時要求補償」這個行為的量化
- NPV 不是計算工具，而是「時間偏好」這個行為特質的表達
- WACC 不是加權平均，而是「有限資本必須被分配」這個行為困境的解法

這很接近財務的三大決策（融資/投資/分配），但「保存」不太對。
**切入角度 C：時間 × 知識 × 資源（這個最有力）**
```

┌──────────────────────────────────────────────────────┐

│          → 「確認」是決策的前提條件                   │

## 溢酬是什麼意思？
漢字拆開來看：
```
溢 = 超出、溢出
酬 = 報酬、補償
溢酬 = 超出基準線的那部分報酬
```
具體來說，政府公債被視為「零風險」，假設給 3%。你投資股票會承擔額外風險，於是要求 8%——多出來的那 5% 就是溢酬，也就是「為了承擔風險而額外要求的補償」。叫「溢」是因為它**溢出**了無風險報酬的基準。
---
## 循環的順序是固定的嗎？
循環是這樣的：
```
獲取 → 衡量 → 配置 → 下一輪獲取
```
這個順序**在邏輯上**是固定的——拿到錢（獲取）之後才能評估機會（衡量），評估完才能決定往哪裡放（配置），放出去產生報酬後成為下一輪可以獲取的資金。
但**在考題裡**，獲取的成本（WACC）通常作為已知條件直接給定，出發點看起來像是從衡量開始。這不是順序錯了，而是題目把「獲取」這個步驟的結果提前告訴你了。
---
## 折現 → 溢酬 → 定價 的順序是固定的嗎？
折現和溢酬**不是嚴格先後**，而是同時被摺進折現率裡：
```
折現率（必要報酬率）= 無風險利率 + 風險溢酬
                         ↑折現       ↑溢酬
```
兩者在計算開始時就已經合體，不需要分兩步執行。定價才是確實在最後——它是把合體後的折現率對著現金流算出來的**結果**。
所以更精確的描述是：
```
（折現 + 溢酬 同時決定折現率）→ 定價（套公式算現值）
```

**k 是什麼？只要折算＋溢酬就一定是 k 嗎？**
k 是必要報酬率（required rate of return），也就是「折現率＋風險溢酬」合體之後的那個數字。凡是要把未來現金流折回現在，就需要一個 k 代入公式，所以「折算＋溢酬」的結果一定是 k。不同管道的 k 有不同的下標：ks（普通股）、kp（特別股）、kd（負債）。

**什麼是 D1？**
D 代表股利（Dividend），右下角的數字代表時間點。D1 是「第一期（明年）預計要發的股利」，D0 是今年已發的股利。Gordon Model 代入的是 D1，因為今天的價格反映的是「未來」現金流的現值。

**什麼是 P0？**
P 代表股票價格（Price），右下角 0 代表「現在（時間點零）」。P0 就是今天的公平股價，也就是要求算的答案。

股票可以看成「每年按 g 比率增長的永久股利流」。數學上，這種「永續等比成長現金流」的現值公式推導出來就是 D1 ÷ (k − g)。直覺解釋：ks 是投資人要求的總報酬，g 是股利自己會成長的部分，兩者相差才是「需要靠今年股利補足的報酬率」，用這個淨報酬率去折算就得出今天的價格。
---
**什麼是 ks？**
ks 是普通股必要報酬率，也就是普通股投資人要求的最低報酬，結構是無風險利率＋風險溢酬。ks 在 Gordon Model 裡同時扮演兩個角色：它既是 L4 裡「折算」的那個 k，也包含了「溢酬」的部分。

**什麼是 g？**
g 是股利固定年成長率（growth rate），假設公司每年股利按同一比率 g 永久增長。Q11 的 g = 4% 代表每年股利比前一年多 4%，這個假設讓無限期的現金流可以用一個簡單公式收斂成有限的現值。

**使用者**：剛剛你漏掉了一個問題，什麼是PV?

**Claude**：PV 是現值（Present Value），也就是「未來某筆錢，在今天值多少」。
核心邏輯是：今天的 100 元比一年後的 100 元有價值，因為今天的錢可以拿去投資、賺取報酬。所以未來的錢必須「折算」回現在，才能和今天的錢做比較。
折算公式：
```
PV = FV ÷ (1 + k)^n
FV：未來的金額（Future Value）
k：折現率（必要報酬率）
n：期數（幾年後）
```
對應到 Q9 公債題：
```
第 1 年息票的 PV = 12,000 ÷ (1 + 0.10)^1
第 2 年息票的 PV = 12,000 ÷ (1 + 0.10)^2
...以此類推，全部加總 = 今天這張公債的公平價格
```
Gordon Model（Q11 特別股和成長股）的 P0，本質上也是 PV——把「未來無限期的股利」全部折算回今天的總和，只是數學上可以化簡成 D1 ÷ (k − g) 這個公式，不需要逐期相加。
簡單說：**PV 是 L4「折算」步驟的輸出結果**，也是定價的最終數字。

## PVIF(k, n)
全名 Present Value Interest Factor，中文叫**現值利率因子**，是「單筆金額」的折現係數：
```
PVIF(k, n) = 1 ÷ (1 + k)^n
```
用途：把第 n 期末的**一筆錢**折算回現在。Q9 裡的本金 100,000 在第 5 年末收回，就用 PVIF(10%, 5) = 0.6209 折算。
---
## PVIFA(k, n)
全名 Present Value Interest Factor of Annuity，中文叫**年金現值利率因子**，是「每期相同金額」的折現係數加總：
```
PVIFA(k, n) = [1 − 1÷(1+k)^n] ÷ k
```
用途：把每期**固定金額**（例如每年 12,000 元的息票）全部折算回現在，不需要逐期算再相加，直接乘以這個係數就能得到加總結果。
---
## F 代表什麼？
F 是 Face Value（面額），也就是債券到期時發行人**歸還的本金**。Q9 裡面額是 100,000 元，在第 5 年末一次還清，所以用 PVIF 折算這一筆。
---
## 完整公式拆解（對照 Q9）
```
P = C × PVIFA(k, n)  +  F × PVIF(k, n)
    ↑                    ↑
    每年息票 12,000       面額 100,000
    × 3.7908             × 0.6209
    = 45,489             = 62,090
    ─────────────────────────────
    合計 ≈ 107,580 → 選 C
```
PVIFA 處理「多筆固定現金流」，PVIF 處理「最後一筆本金」，兩段相加就是整張債券的公平現值。

## K-Constants（k 群，折現率族）
k 群是「折現 + 溢酬」合體後的數值化表達，每個符號對應一個具體的資本來源或用途：
| 符號 | 名稱 | 定義方式 |
|------|------|---------|
| r_f | 無風險利率 | 折現的基準（政府公債殖利率） |
| RP | 風險溢酬 | 超出 r_f 的補償，= β × 市場風險溢酬 |
| k | 必要報酬率（通稱）/ 市場殖利率 | r_f + RP；同一個數字的兩種視角：投資人角度稱「必要報酬率」（要求的最低報酬），市場角度稱「市場殖利率」（資產目前的殖利率） |
| ks | 普通股必要報酬率 | DDM 法：ks = D1 ÷ P0 + g；或 CAPM 法 |
| kd | 稅前負債成本 / 市場殖利率（YTM） | 投資人角度：要求負債給的報酬率；市場角度：債券目前的 YTM；兩個名稱指同一個 k |
| kd(1-T) | 稅後負債成本 | kd × (1 − 稅率 T)，負債有稅盾 |
| kp | 特別股資金成本 | kp = D ÷ P0（無稅盾，不乘 1-T） |
| WACC | 加權平均資金成本 | = Σ（各來源成本 × 權重），配置的門檻 |
| β | 系統風險係數 | 溢酬的輸入參數，衡量相對市場的波動幅度 |
| g | 股利成長率 | DDM 中 ks 的組成部分 |
---
## CF-Constants（CF 群，現金流族）
CF 群是 k 群折現時需要代入的材料，分成輸入和輸出兩類：
| 符號 | 名稱 | 定義 | 角色 |
|------|------|------|------|
| D0 | 本期已發股利 | 今年（時間點零）實際發出的股利 | 輸入 |
| D1 | 下期預計股利 | 明年預計發出的股利 = D0 × (1+g) | 輸入 |
| g | 股利成長率 | 每年股利的固定成長比率（Gordon Model 用）| 輸入 |
| P0 | 目前公平股價 | 今天的合理價格，是 L2 定價步驟的輸出 | 輸出 |
| PV | 現值 | 未來現金流折算到現在的價值 | 輸出 |
| NPV | 淨現值 | PV(未來現金流) − 初始投資，配置決策的核心指標 | 輸出 |
---
**兩群的關係：**
```
CF 群（D1、g）→ 輸入公式 → k 群（ks）折現 → CF 群（P0）輸出
```
k 群決定「用什麼利率折」，CF 群決定「折什麼、折出來是多少」。

---
## 為什麼 k 是分母？
從最基本的現值公式出發：
```
PV = FV ÷ (1 + k)^n
```
k 藏在 `(1 + k)^n` 這一整個式子裡，而這個式子在**分母**的位置。PVIF 只是把這個算式包裝成一個係數：
```
PVIF(k, n) = 1 ÷ (1 + k)^n
```
所以 k 從來都在分母，不是在分子。
---
## 為什麼 k 下降，分母變小，價格上漲？
```
k = 10%  → (1 + 0.10)^5 = 1.6105  → PVIF = 1 ÷ 1.6105 = 0.6209
k =  8%  → (1 + 0.08)^5 = 1.4693  → PVIF = 1 ÷ 1.4693 = 0.6806
```
k 下降 → (1+k)^n 變小 → 整個分母變小 → 同一筆未來現金流折算出更高的現值。
---
## 溢酬是「超過折現率」還是「超過無風險利率」？
溢酬（RP，Risk Premium）是超過**無風險利率 r_f** 的補償，不是超過折現率：
```
k（折現率）= r_f（無風險利率）+ RP（風險溢酬）
             ↑                    ↑
             折算的基準            額外的風險補償
```
折算和溢酬是 k 的**組成部分**，k 本身就是兩者合體後的結果。說「溢酬超過折現率」的邏輯會變成循環——折現率就是 k，k 包含了溢酬，所以溢酬不能同時又超過 k。

**使用者**：什麼是PVIFA?什麼是PVIF?為什麼要算息票PV和本金PV?這兩者有何不同?PVIFA和PVIF有何不同?

**Claude**：逐一回答：
---
## 為什麼要分開算息票 PV 和本金 PV？
因為兩筆現金流的**性質不同**：
```
息票（C）：每年都有一筆，共 n 次，每次金額相同
             → 這是「重複出現的固定金額」
本金（F）：只在第 n 年末出現一次
             → 這是「單筆、只出現一次的金額」
```
數學上無法用同一個公式同時處理「重複多筆」和「單筆」，所以必須拆開計算再加總。
---
## PVIF 是什麼？
專門處理**單筆**未來金額的折現係數：
```
PVIF(k, n) = 1 ÷ (1 + k)^n
用途：第 n 年末的一筆錢，今天值多少
例：PVIF(10%, 3) = 1 ÷ (1.10)^3 = 0.7513
    代表三年後的 1 元，今天只值 0.7513 元
```
---
## PVIFA 是什麼？
專門處理**每期固定金額（年金）** 的折現係數加總：
```
PVIFA(k, n) = PVIF(k,1) + PVIF(k,2) + ... + PVIF(k,n)
            = [1 − 1÷(1+k)^n] ÷ k
用途：未來 n 年、每年固定一筆 C，今天全部加起來值多少
例：PVIFA(10%, 3) = 2.4869
    代表「未來三年每年 1 元」，今天加總值 2.4869 元
```
PVIFA 只是把 n 個 PVIF 全部加總後包裝成一個係數，省去逐期手算的步驟。
---
## PVIF 和 PVIFA 的核心差異
| 比較點 | PVIF | PVIFA |
|--------|------|-------|
| 處理的現金流 | 單筆（只出現一次）| 年金（每期重複相同金額）|
| 計算邏輯 | 折現一次 | 折現 n 次再加總 |
| 對應 Q7 的角色 | 折算第 3 年末的本金 $1,000 | 折算第 1、2、3 年的息票各 $80 |
---
完整公式對照 Q7：
```
P = C × PVIFA(10%, 3)  +  F × PVIF(10%, 3)
    80 × 2.4869             1,000 × 0.7513
    = 198.95                = 751.30
    ─────────────────────────────────
    合計 = 950.25
```

專門處理**單筆**未來金額的折現係數：



```

用途：第 n 年末的一筆錢，今天值多少

共同特徵是分母裡有「兩個力量互相抵銷」，最終的淨值決定價格：
```
Q11 Gordon Model：
  分母 = ks − g
  折現力量（ks）往下拉 vs 成長力量（g）往上頂
  兩力相減後的淨差，才是「水庫實際外流量」
Ch06 Q1（kp）：
  分母 = P0 × (1 − F%)
  市價（P0）是蓄水量，發行成本（F%）是漏水量
  扣掉漏水後，公司實際拿到的才是有效分母
```
兩題的解法模式一樣：**先算出淨分母 → 用單一現金流 D 除以它**。
---
### 多種現金流組合
對應：**Ch05 問答 Q7（政府公債定價）**
特徵是現金流有**兩種不同性質**，無法用同一個係數處理，必須拆開再加總：
```
息票 C × n 次  → 年金性質 → × PVIFA（年金係數）
本金 F，第 n 年末 → 單筆性質 → × PVIF（單筆係數）
                              ↓
                         兩段分別算出 PV，再相加
```
---
### 兩種題型的一眼識別法
| 識別點 | 水庫平衡 | 多種現金流組合 |
|--------|---------|--------------|
| 現金流幾種 | 一種（只有 D） | 兩種（C + F）|
| 分母結構 | 兩個力量相減（ks−g 或 P_net）| 沒有分母結構，靠係數表 |
| 解題步驟 | 算淨分母 → 直接除 | 拆開 → 各自乘係數 → 加總 |

```

   年 2：100 ÷ (1.03)² =  94.26
